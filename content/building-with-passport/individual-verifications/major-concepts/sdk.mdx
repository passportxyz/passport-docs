# SDK Integration

The Human ID SDK allows you to embed Individual Verifications directly into your website or dApp, providing a seamless verification experience without redirecting users.

## Installation

```bash
npm install @holonym-foundation/human-id-sdk
```

## Quick Start

```typescript
import { initHumanID } from '@holonym-foundation/human-id-sdk'

// Initialize the SDK (client-side only, requires window)
const humanID = initHumanID()

// Request verification
const result = await humanID.requestSBT('kyc')        // Government ID (shows selector: KYC or NFC Passport)
// or: humanID.requestSBT('phone')                    // Phone
// or: humanID.requestSBT('biometrics')               // Biometrics
// or: humanID.requestSBT('clean-hands')              // Proof of Clean Hands

if (result?.recipient) {
  console.log('Verified! Attestation issued to:', result.recipient)
}
```

<Callout type="info">
**Government ID method selector**: Calling `requestSBT('kyc')` now presents users with a choice between traditional KYC (~$5, any government ID) and NFC Passport verification (free, requires biometric passport with NFC chip). Both methods share the same sybil resistance — a user verified via either method will be recognized by the `gov-id` API endpoint.
</Callout>

## Integration Options

| Use Case | Recommendation |
|----------|----------------|
| Client-side dApps | SDK - embeds verification flow directly |
| Server-side checks | [API](/building-with-passport/individual-verifications/api-reference) - simple true/false queries |
| On-chain contracts | [Attestation Protocols](/building-with-passport/individual-verifications/major-concepts/attestation-protocols) - query Sign Protocol directly |
